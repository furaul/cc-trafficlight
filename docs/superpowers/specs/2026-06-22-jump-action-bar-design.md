# 设计：探出动作条 + 一键/快捷键跳转

日期：2026-06-22
状态：已与用户确认交互，待评审

## 背景与目标

现有药丸（`app/ui/widget.js`）需要先点击才能展开会话列表，再点列表行才能跳转。当某个会话**需要确认（红）或刚完成（绿）**时，用户希望**少点一次**：提醒一出现就能一键跳到该 tab；并且在红框/绿框闪烁时，能用**全局快捷键**直接跳到需要响应的 tab，而不必把手从键盘移到鼠标。

## 已确认的交互决策

1. **触发状态**：`waiting`（红，待确认）+ **刚完成的** `idle`（绿，已完成待查看）。不含 `working`/`attention`。
2. **多个待响应时**：动作条**只显示最紧急的一个**（红 > 绿，同级按 ⌘N 升序），其余用 **`+N`** 计数。
3. **快捷键**：单一全局快捷键 **⌥⌘J**，每按一次跳到下一个待响应 tab，在所有待响应会话间循环（红优先，再绿）。
4. **运行时长**：动作条显示该状态已持续的时长（如 `2m`）。
5. **绿框强度**：维持柔和脉冲（比红框弱），已确认合适。
6. **自动收起**：点击跳转后、或该会话不再处于待响应状态时，动作条自动收起。

## 交互细节

### 探出动作条（peek）

- 位置：右下角药丸**上方**。即使药丸处于收起态，只要存在待响应会话就自动「探出」（带弹性长高动画），无需先点开列表。
- 内容（单行）：`● 状态色圆点` ＋ `项目名 ⌘N` ＋ `状态文字 · 运行时长` ＋ 右侧 `跳转 →` 按钮；右上角 `+N`（当待响应数 > 1）。
- 红色项的圆点闪烁；绿色项常亮。
- 整条可点击 = 跳转到该 tab（复用现有 `cc_jump_index`）。

### 最紧急项的选取

待响应集合 = `state==waiting` 或（`state==idle` 且「刚完成」）。
排序：`waiting`(优先级2) > `idle`(优先级1)；同级按 `(win, tab)` 升序。动作条显示集合首位。

**「刚完成」的判定**：复用现有 `updatedAt`。会话转入 `idle` 时 `updatedAt` 即更新；定义常量 `DONE_PEEK_TTL`（默认 **300 秒**）——`idle` 且 `now - updatedAt < DONE_PEEK_TTL` 才算「刚完成、待查看」，进入待响应集合；超时后该会话仍留在完整列表里，但不再触发动作条与绿框。`waiting` 无 TTL，持续到状态改变。

### 全局快捷键 ⌥⌘J

- 用 `tauri-plugin-global-shortcut` 注册系统级快捷键，App 失焦时也生效。
- 触发时 Rust 端 `app.emit("hotkey-cycle")`；前端 `widget.js` 监听该事件，在「待响应集合」中按游标取下一个，调用 `cc_jump_index(win, tab)` 跳转，游标 +1 循环。
  - 选此方案的原因：tab↔会话的匹配逻辑（`buildAssignments`，依赖 Ghostty AX 标题）已在前端，且 webview 在后台常驻运行；让前端处理循环可复用全部现有匹配代码，避免在 Rust 里重复实现一套。
- 集合为空时：什么都不跳（可选轻提示）。

### 跳转后的状态流转

- 跳转到**绿（已完成）**：视为已查看 → 该项移出待响应集合（动作条切到下一个或收起）。
- 跳转到**红（待确认）**：仅切到该 tab，状态不变；用户在该 tab 实际响应后，会话自然转 `working`，动作条随之收起。

### 全屏边框联动

- 有任一 `waiting` → 红框脉冲（现有逻辑）。
- 无红、但有「刚完成」绿 → 绿框柔和脉冲。
- 两者皆无 → 不闪。

## 涉及改动（实现层面，概要）

- **`app/src-tauri/src/state.rs`**：新增「待响应」判定（`waiting` 或 `idle && age<DONE_PEEK_TTL`）的辅助逻辑 + 单测；`DONE_PEEK_TTL` 常量。
- **`app/src-tauri/src/main.rs`**：集成 `tauri-plugin-global-shortcut`，注册 ⌥⌘J，触发时 emit `hotkey-cycle`。
- **`app/src-tauri/Cargo.toml` / `capabilities/default.json`**：加 global-shortcut 插件与权限。
- **`app/ui/widget.js`**：
  - 渲染探出动作条（收起态也显示）：最紧急项 + `+N` + 运行时长。
  - 监听 `hotkey-cycle`，在待响应集合上循环 `cc_jump_index`。
  - 跳转绿项后将其移出待响应（本地标记已查看，直到状态再变）。
- **`app/ui/widget.html` / `common.css`**：动作条样式与展开/收起动画。

## 不做（YAGNI）

- 快捷键自定义配置（v1 硬编码 ⌥⌘J，后续再做）。
- 检测「用户手动切到了该 tab」来消绿（v1 用 TTL + 点击跳转两种消解方式即可）。
- 动作条里堆叠展示多个（已确认只显最紧急 + 计数）。
